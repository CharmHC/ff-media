# M7 PR 2 — Video Merger UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the M7 merge engine a UI — a `MergerPage` where the user adds clips, orders them (manually, randomly, or randomly with clips locked to an index), reviews the auto-derived standardization target, sees the output duration and a merge-time estimate, then merges with live progress and cancel.

**Architecture:** The engine already exists and is unchanged in shape (PR 1, merged): `IMediaAnalyzer` probes files, `MergeTargetDerivation` proposes a target, `ConformanceCheck` says which clips need re-encoding, `MergeEstimator` prices the job, and `IMergeService` runs it. This PR adds `MergerViewModel` (headless, unit-tested with fakes) plus a thin `MergerPage.xaml`, registers a `VideoMergerTool : ITool` so the shell picks it up **without being modified**, and wires the existing history/notification/settings services. Two small, deliberate engine changes are in scope (Tasks 1–2): per-clip progress percentages, and a `Source` field on `HistoryEntry` so a merge is a first-class history row.

**Tech Stack:** C# / .NET 9, WPF + WPF-UI 4.3.0, CommunityToolkit.Mvvm (`ObservableObject`, `[ObservableProperty]`, `[RelayCommand]`), Microsoft.Extensions.DependencyInjection, xUnit (plain `Assert`).

## Decisions taken before this plan (do not re-litigate)

1. **Per-clip progress bars** (user-chosen). The engine's `MergeProgress` reported only an overall percent and the name of the clip currently encoding. Task 1 extends it with per-clip percentages. This reopens PR 1's `MergeService`/`ProgressTracker` deliberately.
2. **`HistoryEntry` gains a `Source`** (user-chosen), so a merge is a first-class row rather than a download with a blank URL. Task 2. Old `history.json` files must still load.
3. **`SortOrder = 20`, not the spec's `2`.** The spec says `SortOrder = 2` meaning "the second tool", but ordering is **ascending** and `YouTubeDownloaderTool.SortOrder` is `10` — so `2` would put the merger *above* the downloader, inverting the spec's own intent. Use `20`.
4. **No `IErrorMapper`.** Spec §8 cites one; it has never existed in this codebase. The real convention is a static per-module mapper (`YtDlpErrors`). Task 3 adds `MergeErrors` in the same shape.

- [ ] **Step 4: Teach `ProgressTracker` to publish it**

In `src/FFMedia.Tools.VideoMerger/Services/MergeService.cs`, the tracker is constructed with the *work* slots only. It now also needs to know how many clips there are in total and which clip index each slot maps to. Change the constructor and add the snapshot builder:

```csharp
private sealed class ProgressTracker
{
    private readonly IProgress<MergeProgress>? _sink;
    private readonly double[] _fractions;
    private readonly double[] _weights;
    private readonly int[] _slotToClipIndex;
    private readonly int _clipCount;
    private readonly double _encodeWeight;
    private readonly Lock _gate = new();
    private double _highWater;

    public ProgressTracker(
        IProgress<MergeProgress>? sink,
        double[] clipSeconds,
        int[] slotToClipIndex,
        int clipCount,
        double encodeWeight)
    {
        _sink = sink;
        _encodeWeight = encodeWeight;
        _slotToClipIndex = slotToClipIndex;
        _clipCount = clipCount;
        _fractions = new double[clipSeconds.Length];
        _weights = new double[clipSeconds.Length];

        var total = clipSeconds.Where(s => s > 0).Sum();
        for (var i = 0; i < clipSeconds.Length; i++)
        {
            // Zero/unknown durations fall back to an equal share — better a slightly wrong bar
            // than a division by zero or a clip that contributes nothing when it completes.
            _weights[i] = total > 0 && clipSeconds[i] > 0
                ? clipSeconds[i] / total
                : 1.0 / clipSeconds.Length;
        }
    }

    /// <summary>Per-clip percentages in request order. Any clip without a normalize slot already
    /// conforms, so it is 100 — it is not waiting on work, it simply has none.</summary>
    private double[] ClipPercents()
    {
        var percents = new double[_clipCount];
        Array.Fill(percents, 100.0);
        for (var slot = 0; slot < _slotToClipIndex.Length; slot++)
        {
            percents[_slotToClipIndex[slot]] = Math.Clamp(_fractions[slot] * 100.0, 0, 100);
        }

        return percents;
    }
}
```

Every `Emit(...)` inside the tracker must now pass `ClipPercents()` into the `MergeProgress` it constructs. Find `Emit` and thread it through; do **not** compute the array outside the lock.

At the call site in `MergeAsync`, the tracker is built from `plan.Work`. Supply the mapping:

```csharp
var tracker = new ProgressTracker(
    progress,
    [.. plan.Work.Select(w => w.Clip.Info.Duration.TotalSeconds)],
    [.. plan.Work.Select(w => w.Index)],
    request.Clips.Count,
    plan.Work.Count == 0 ? 0 : EncodeWeight);
```

`MergePlan.Work` is a `List<(int Index, MergeClip Clip)>` — `Index` is already the clip's position in `MergeRequest.Clips`, which is exactly the mapping needed.

```csharp
namespace FFMedia.Tools.VideoMerger.Services;

/// <summary>Turns the ffmpeg stderr tail that <c>FfmpegRunner</c> captures into something a user
/// can act on. Anything unrecognized passes through verbatim — a generic "merge failed" would
/// destroy the only diagnostic the user, or a bug report, has to go on.</summary>
public static class MergeErrors
{
    public static string Describe(string ffmpegError)
    {
        if (string.IsNullOrWhiteSpace(ffmpegError))
        {
            return "The merge failed, but ffmpeg reported no reason.";
        }

        if (Contains(ffmpegError, "no space left"))
        {
            return "The disk filled up during the merge. Free some space and try again.";
        }

        if (Contains(ffmpegError, "invalid data found"))
        {
            return "One of the clips is corrupt or is not a video file FFMedia can read.";
        }

        if (Contains(ffmpegError, "permission denied"))
        {
            return "FFMedia could not write the output file. Check the folder's permissions, "
                + "or pick another folder.";
        }

        if (Contains(ffmpegError, "unknown encoder"))
        {
            return "This ffmpeg build cannot encode the chosen video codec. Pick H.264, "
                + "or re-run build/fetch-binaries.ps1.";
        }

        return ffmpegError;
    }

    private static bool Contains(string haystack, string needle)
        => haystack.Contains(needle, StringComparison.OrdinalIgnoreCase);
}
```

```csharp
using System.IO;
using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;

/// <summary>The Video Merger page's brain. Headless and fully unit-testable: every dependency is
/// an interface, and recomputation (conformance + estimate) is synchronous and cheap — no probe is
/// ever repeated once a clip is in the list.</summary>
public partial class MergerViewModel : ObservableObject
{
    private readonly IMediaAnalyzer _analyzer;
    private readonly IMergeService _merger;
    private readonly ISpeedProfileStore _speeds;
    private readonly IHistoryService _history;
    private readonly INotificationService _notifications;

    public MergerViewModel(
        IMediaAnalyzer analyzer,
        IMergeService merger,
        ISpeedProfileStore speeds,
        ISettingsService settings,
        IHistoryService history,
        INotificationService notifications)
    {
        ArgumentNullException.ThrowIfNull(analyzer);
        ArgumentNullException.ThrowIfNull(merger);
        ArgumentNullException.ThrowIfNull(speeds);
        ArgumentNullException.ThrowIfNull(settings);
        ArgumentNullException.ThrowIfNull(history);
        ArgumentNullException.ThrowIfNull(notifications);

        _analyzer = analyzer;
        _merger = merger;
        _speeds = speeds;
        _history = history;
        _notifications = notifications;

        OutputFolder = settings.Current.DefaultOutputFolder;
        Clips.CollectionChanged += (_, _) => Recompute();
    }

    public ObservableCollection<MergeClipViewModel> Clips { get; } = [];

    /// <summary>Seeds the shuffle. Exposed so tests are deterministic; the UI re-seeds it from the
    /// clock on every Shuffle click.</summary>
    public int ShuffleSeed { get; set; } = Environment.TickCount;

    [ObservableProperty] private string _outputFolder;

    [RelayCommand]
    public async Task AddClipsAsync(IEnumerable<string> paths)
    {
        ArgumentNullException.ThrowIfNull(paths);

        foreach (var path in paths)
        {
            if (Clips.Any(c => string.Equals(c.SourcePath, path, StringComparison.OrdinalIgnoreCase)))
            {
                continue; // already in the list — do not probe it again
            }

            var probe = await _analyzer.AnalyzeAsync(path).ConfigureAwait(true);
            if (!probe.IsSuccess || probe.Value!.Video is null)
            {
                // Reject at add-time (spec §8): a file that cannot be merged must never reach the
                // list, or the whole merge fails later — after the user has ordered everything.
                _notifications.Notify(new Notification(
                    "Not a video",
                    $"{Path.GetFileName(path)} could not be read as a video and was not added.",
                    NotificationSeverity.Warning));
                continue;
            }

            Clips.Add(new MergeClipViewModel(new MergeClip(path, probe.Value)));
        }

        Recompute();
    }

    [RelayCommand]
    public void RemoveClip(MergeClipViewModel clip)
    {
        ArgumentNullException.ThrowIfNull(clip);
        Clips.Remove(clip);
        ResyncLocks();
    }

    [RelayCommand]
    public void MoveUp(MergeClipViewModel clip)
    {
        ArgumentNullException.ThrowIfNull(clip);
        var index = Clips.IndexOf(clip);
        if (index > 0)
        {
            Clips.Move(index, index - 1);
            ResyncLocks();
        }
    }

    [RelayCommand]
    public void MoveDown(MergeClipViewModel clip)
    {
        ArgumentNullException.ThrowIfNull(clip);
        var index = Clips.IndexOf(clip);
        if (index >= 0 && index < Clips.Count - 1)
        {
            Clips.Move(index, index + 1);
            ResyncLocks();
        }
    }

    [RelayCommand]
    public void Shuffle()
    {
        if (Clips.Count < 2)
        {
            return;
        }

        var shuffled = Ordering.Shuffle([.. Clips], c => c.LockedIndex, ShuffleSeed);

        // Rebuild in place: assigning through the ObservableCollection keeps the bound ListView's
        // selection and virtualization sane, and fires exactly one change per moved row.
        for (var i = 0; i < shuffled.Count; i++)
        {
            var current = Clips.IndexOf(shuffled[i]);
            if (current != i)
            {
                Clips.Move(current, i);
            }
        }

        ResyncLocks();
    }

    /// <summary>A locked row is pinned to the index it currently occupies. Removing or moving a
    /// *neighbour* shifts it, so re-capture every lock's index after any structural change —
    /// otherwise Ordering.Shuffle would pin a row to a stale slot, or two rows to the same one.</summary>
    private void ResyncLocks()
    {
        for (var i = 0; i < Clips.Count; i++)
        {
            if (Clips[i].IsLocked)
            {
                Clips[i].SetLock(locked: true, index: i);
            }
        }
    }

    /// <summary>Filled in by Task 6.</summary>
    private void Recompute()
    {
    }
}
```

**Rules:**
- One merge at a time (spec D8). `MergeCommand` is disabled while merging; `CancelCommand` only enabled while merging.
- Progress arrives on a **worker** thread. The ViewModel is the one correct place to marshal to the UI thread — but do **not** use the BCL `Progress<T>`. Use a synchronous `IProgress<T>` and let the WPF binding layer handle it: `ObservableObject.SetProperty` raises `PropertyChanged` on the calling thread, and WPF marshals a **scalar** property change automatically. The `ObservableCollection` itself is never mutated during a merge, only each row's `Percent`, so this is safe. State that reasoning in a comment — it is the kind of thing a future reader will "fix" into a deadlock.
- **Cancel is not failure.** A canceled merge notifies `Information`, writes **no** history row, and says "Merge canceled." A failed merge notifies `Error` with `MergeErrors.Describe(...)` and writes no history row either. Only success writes history.

```csharp
    private CancellationTokenSource? _cancellation;

    [ObservableProperty] private bool _isMerging;

    [ObservableProperty] private double _overallPercent;

    [ObservableProperty] private string _statusMessage = "";

    partial void OnIsMergingChanged(bool value)
    {
        OnPropertyChanged(nameof(CanMerge));
        MergeCommand.NotifyCanExecuteChanged();
        CancelCommand.NotifyCanExecuteChanged();
    }

    [RelayCommand(CanExecute = nameof(CanMerge))]
    private async Task MergeAsync()
    {
        _cancellation = new CancellationTokenSource();
        IsMerging = true;
        OverallPercent = 0;
        StatusMessage = "Preparing…";

        foreach (var clip in Clips)
        {
            clip.Percent = 0;
        }

        try
        {
            var request = new MergeRequest([.. Clips.Select(c => c.Clip)], Target, OutputPath);

            // A synchronous IProgress, NOT the BCL Progress<T> — that one posts to the captured
            // SynchronizationContext, so reports arrive out of order and headless tests race.
            // Reporting straight through is safe here because every property touched is a SCALAR:
            // WPF marshals a scalar PropertyChanged to the UI thread for us. The Clips collection
            // itself is never mutated mid-merge. Do not "fix" this into a Dispatcher.Invoke.
            var sink = new SyncProgress<MergeProgress>(OnMergeProgress);

            var result = await _merger.MergeAsync(request, sink, _cancellation.Token)
                .ConfigureAwait(true);

            if (result.IsSuccess)
            {
                OverallPercent = 100;
                StatusMessage = "Merge complete.";
                _history.Append(new HistoryEntry(
                    Title: OutputFileName,
                    Url: "",                       // a merge has no URL — its inputs are local files
                    OutputPath: result.Value,
                    Format: DescribeTarget(),
                    Timestamp: DateTimeOffset.Now,
                    Status: "Completed",
                    Source: HistorySource.Merge));
                _notifications.Notify(new Notification(
                    "Merge complete", $"Saved to {result.Value}", NotificationSeverity.Success));
                return;
            }

            // Cancellation is not failure. The engine reports it as a failure Result, but telling a
            // user who just clicked Cancel that their merge "broke" is a lie, and a red toast for an
            // action they took on purpose is worse than none.
            if (_cancellation.IsCancellationRequested)
            {
                StatusMessage = "Merge canceled.";
                _notifications.Notify(new Notification(
                    "Merge canceled", "Merge canceled.", NotificationSeverity.Information));
                return;
            }

            var friendly = MergeErrors.Describe(result.Error!);
            StatusMessage = friendly;
            _notifications.Notify(new Notification("Merge failed", friendly, NotificationSeverity.Error));
        }
        finally
        {
            IsMerging = false;
            _cancellation.Dispose();
            _cancellation = null;
        }
    }

    private bool CanCancel => IsMerging;

    [RelayCommand(CanExecute = nameof(CanCancel))]
    private void Cancel() => _cancellation?.Cancel();

    private void OnMergeProgress(MergeProgress progress)
    {
        OverallPercent = progress.OverallPercent;
        StatusMessage = progress.Status switch
        {
            MergeJobStatus.Normalizing => progress.CurrentClip is null
                ? "Standardizing clips…"
                : $"Standardizing {progress.CurrentClip}…",
            MergeJobStatus.Concatenating => "Joining clips…",
            _ => StatusMessage,
        };

        for (var i = 0; i < Clips.Count && i < progress.ClipPercents.Count; i++)
        {
            Clips[i].Percent = progress.ClipPercents[i];
        }
    }

    private string DescribeTarget()
        => string.Create(
            CultureInfo.InvariantCulture,
            $"{Target.Container.ToString().ToUpperInvariant()} · {Target.Width}x{Target.Height} · {Target.FrameRate.Value:0.###} fps");

    /// <summary>Reports on the calling thread. See the comment in MergeAsync for why this is right
    /// and the BCL Progress&lt;T&gt; is not.</summary>
    private sealed class SyncProgress<T> : IProgress<T>
    {
        private readonly Action<T> _handler;
        public SyncProgress(Action<T> handler) => _handler = handler;
        public void Report(T value) => _handler(value);
    }
```

```csharp
using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;
using System.Threading.Tasks;
using FFMedia.Core.Binaries;
using FFMedia.Core.Processes;
using FFMedia.Media;
using FFMedia.Tools.VideoMerger.Models;
using FFMedia.Tools.VideoMerger.Services;
using Microsoft.Extensions.Logging.Abstractions;
using Xunit;

namespace FFMedia.Tests.Integration;

[Trait("Category", "Integration")]
public class MergeIntegrationTests : IDisposable
{
    private readonly string _dir = Path.Combine(Path.GetTempPath(), "ffmedia-merge-it-" + Guid.NewGuid().ToString("N"));

    public MergeIntegrationTests() => Directory.CreateDirectory(_dir);

    public void Dispose()
    {
        if (Directory.Exists(_dir))
        {
            Directory.Delete(_dir, recursive: true);
        }

        GC.SuppressFinalize(this);
    }

    private static IBinaryProvider Binaries()
        => new BundledBinaryProvider(Path.Combine(AppContext.BaseDirectory, "assets", "binaries"));

    /// <summary>Synthesizes a clip with ffmpeg's own testsrc, at a chosen size and rate, so the
    /// merge has genuinely mismatched inputs to standardize.</summary>
    private async Task<string> MakeClipAsync(string name, string size, int fps, int seconds)
    {
        var path = Path.Combine(_dir, name);
        var runner = new ProcessRunner();
        var result = await runner.RunAsync(Binaries().GetPath(ExternalBinary.Ffmpeg), [
            "-hide_banner", "-nostdin", "-y",
            "-f", "lavfi", "-i", $"testsrc=size={size}:rate={fps}:duration={seconds}",
            "-f", "lavfi", "-i", $"sine=frequency=440:duration={seconds}",
            "-c:v", "libx264", "-pix_fmt", "yuv420p", "-c:a", "aac", "-shortest", path,
        ]);

        Assert.Equal(0, result.ExitCode);
        return path;
    }

    [Fact]
    public async Task MergeAsync_JoinsThreeMismatchedClips_IntoOneStandardizedFile()
    {
        var runner = new ProcessRunner();
        var binaries = Binaries();
        var analyzer = new FfprobeMediaAnalyzer(runner, binaries);

        // Deliberately mismatched: different resolutions AND different frame rates.
        var paths = new[]
        {
            await MakeClipAsync("a.mp4", "1280x720", 30, 2),
            await MakeClipAsync("b.mp4", "1920x1080", 25, 2),
            await MakeClipAsync("c.mp4", "640x480", 60, 2),
        };

        var clips = new List<MergeClip>();
        foreach (var path in paths)
        {
            var probe = await analyzer.AnalyzeAsync(path);
            Assert.True(probe.IsSuccess, probe.Error);
            clips.Add(new MergeClip(path, probe.Value!));
        }

        var target = MergeTargetDerivation.Derive([.. clips.Select(c => c.Info)]);
        var output = Path.Combine(_dir, "merged.mp4");

        var service = new MergeService(
            new FfmpegRunner(runner, binaries),
            new SpeedProfileStore(_dir, NullLogger<SpeedProfileStore>.Instance),
            _ => long.MaxValue,      // disk space is not what this test is about
            _dir,
            maxConcurrency: 2,
            NullLogger<MergeService>.Instance);

        var result = await service.MergeAsync(new MergeRequest(clips, target, output));

        Assert.True(result.IsSuccess, result.Error);
        Assert.True(File.Exists(output));

        // The real proof: probe the OUTPUT. It must be one file, ~6s, at the derived target.
        var merged = await analyzer.AnalyzeAsync(output);
        Assert.True(merged.IsSuccess, merged.Error);

        var info = merged.Value!;
        Assert.Equal(6, info.Duration.TotalSeconds, precision: 0); // 3 x 2s, no transitions
        Assert.Equal(target.Width, info.Video!.Width);
        Assert.Equal(target.Height, info.Video.Height);
        Assert.Equal("h264", info.Video.CodecName);
        Assert.Equal("yuv420p", info.Video.PixelFormat);
        Assert.True(info.HasAudio);
        Assert.Equal(0, info.ExtraStreamCount);

        // And no temp debris survived.
        Assert.Empty(Directory.GetDirectories(_dir, "merge-*"));
    }

    [Fact]
    public async Task MergeAsync_FastPath_StreamCopiesIdenticalClips()
    {
        var runner = new ProcessRunner();
        var binaries = Binaries();
        var analyzer = new FfprobeMediaAnalyzer(runner, binaries);

        // Same size, same rate -> every clip already conforms -> no re-encode at all.
        var paths = new[]
        {
            await MakeClipAsync("x.mp4", "1280x720", 30, 2),
            await MakeClipAsync("y.mp4", "1280x720", 30, 2),
        };

        var clips = new List<MergeClip>();
        foreach (var path in paths)
        {
            var probe = await analyzer.AnalyzeAsync(path);
            clips.Add(new MergeClip(path, probe.Value!));
        }

        var target = MergeTargetDerivation.Derive([.. clips.Select(c => c.Info)]);
        Assert.All(clips, c => Assert.True(ConformanceCheck.Evaluate(c.Info, target).IsConforming));

        var output = Path.Combine(_dir, "fast.mp4");
        var service = new MergeService(
            new FfmpegRunner(runner, binaries),
            new SpeedProfileStore(_dir, NullLogger<SpeedProfileStore>.Instance),
            _ => long.MaxValue, _dir, 2, NullLogger<MergeService>.Instance);

        var result = await service.MergeAsync(new MergeRequest(clips, target, output));

        Assert.True(result.IsSuccess, result.Error);
        var merged = await analyzer.AnalyzeAsync(output);
        Assert.Equal(4, merged.Value!.Duration.TotalSeconds, precision: 0);
    }
}
```
